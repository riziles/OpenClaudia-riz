### Added
- ACP IDE-bridge notifications (crosslink #517). Editor plugins can now push `ide/file_opened`, `ide/file_closed`, `ide/selection_changed`, and `ide/diagnostics` JSON-RPC notifications; the server maintains an `IdeState` snapshot (active file, most-recent-first capped recents ring, current selection with file+range+text, per-file LSP diagnostics) and exposes it via `AcpServer::ide_state()` so the prompt builder can inject editor context on the next turn. Message schemas match Claude Code's MCP notifications field-for-field. Malformed payloads are logged-and-dropped rather than surfaced as errors — one bad notification must not crash the bridge loop. 6 new tests
- compact-boundary markers (crosslink #516 partial). `compaction::build_compact_boundary_message` now inserts a dedicated system message with a stable sentinel prefix + JSON metadata (trigger / pre_tokens / messages_summarized) directly before the summary, matching Claude Code's `createCompactBoundaryMessage` shape. New predicates `is_compact_boundary_message` / `extract_compact_boundary_metadata`. `transcript::entries_after_last_boundary` gives `--resume` a single-call way to skip pre-boundary history so the model never re-sees content that was already summarized away. 6 new tests
- per-session task-list storage (crosslink #518). `todo_write` / `todo_read` now bucket by session via a thread-local `CURRENT_SESSION_ID` set through the `SessionIdGuard` RAII wrapper — drop semantics guarantee the `tokio::spawn_blocking` worker thread that runs each tool sees a clean slate afterward, so the next unrelated tool on the same worker can't read a leaked session key. TUI / chat REPL / subagent executor all wrap calls with a guard (subagent uses its `agent_id`, matching Claude Code's `agentId ?? sessionId` fallback). Unbucketed callers (tests, pre-session REPL) fall through to a `__default__` key instead of colliding with real sessions. 2 new tests
- hook trigger points: `PostToolUse` / `PostToolUseFailure` now fire after every tool execution in the TUI pipeline; `SessionEnd` fires on app exit; `Stop` fires at each terminal assistant response; `Notification` fires on API errors (crosslink #513). New `HookEngine::fire_post_tool` helper. `PostToolUseFailure` with no dedicated handlers falls back to `PostToolUse` — matches Claude Code's single-handler-sees-both behavior. Full coverage of CC settings.json hook keys in the merge layer (`PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `UserPromptSubmit`, `SessionStart`, `SessionEnd`, `Stop`, `SubagentStart`, `SubagentStop`, `PreCompact`, `PermissionRequest`, `Notification`). 3 new tests
- tool parity: closed four Claude Code parity gaps in existing tools (crosslink #509). `todo_write` now auto-clears the list when every item is `completed` (matches CC's `allDone` branch). `notebook_edit` accepts Claude Code's stable `cell_id` parameter, with `cell_number` kept as a back-compat alias — for `insert`, `cell_id` means "insert AFTER this cell" (CC semantics), unknown IDs return a clear error. `web_search` accepts `allowed_domains` / `blocked_domains` lists with hostname-suffix matching (`"docs.python.org"` covers `foo.docs.python.org`) and a 2-char query minimum. `ask_user_question` adds `multiSelect` (canonical CC name; legacy `multi_select` still accepted and auto-normalized), optional `preview` per option, and CC's uniqueness refinement (question texts unique, option labels unique within a question). 13 new tests
- migrations: new `src/migrations/` framework ports Claude Code's on-disk migration pattern. Each migration is an idempotent or once-only unit (via `Migration` trait + `RunPolicy`) that the runner invokes before any subsystem touches the stores it manages. Once-only bookkeeping lives at `~/.local/share/openclaudia/migrations.json`. Runner never aborts startup on failure — per-migration outcomes surface via `run_all`'s return value. First real migration (`stamp-transcript-schema-v1`) writes a baseline `projects/.schema-version.json` so future transcript-format changes have a starting point to diff against. Closes crosslink #506
- thinking: port Claude Code's `ultrathink` / `think ultra hard` / `think ultrahard` keyword detection + `MAX_THINKING_TOKENS` / `CLAUDE_CODE_EFFORT_LEVEL` env precedence. New `src/thinking.rs` module. Keyword in any user message bumps the turn's effort to `high`, which in turn raises the Anthropic `thinking.budget_tokens` to Claude Code's ULTRATHINK constant (31999). Added `max` effort level (alias of `high` with the same budget)
- thinking: effort level now propagates to the OpenAI (`reasoning_effort: "high"` on high/max) and Google Gemini (`generationConfig.thinkingConfig.thinkingBudget`, clamped to Gemini's 24k cap) request builders
- transcripts: new `src/transcript.rs` module writes each session as a Claude Code-compatible JSONL at `$CLAUDE_CONFIG_HOME_DIR/projects/<sanitized-cwd>/<session-id>.jsonl`. `CLAUDE_CONFIG_HOME_DIR` defaults to `~/.claude`. Path sanitization (`[^a-zA-Z0-9]` → `-`) matches Claude Code exactly, so transcripts written here are readable by Claude Code and vice versa. Each line is a `SerializedMessage` envelope (`type`, `uuid`, `timestamp`, `cwd`, `sessionId`, `version`, optional `gitBranch`, `message`). Append-only with `O_APPEND`, 0o600/0o700 perms
- TUI: every turn now dual-writes — the existing JSON snapshot at `~/.local/share/openclaudia/chat_sessions/<uuid>.json` stays for backward compat, and new entries also append to the JSONL transcript via `App::persist_transcript_tail`. Watermark tracking skips re-appending on resume
